iPrint.Direct

What is this?

Direct printing, without interruption or confirmation, direct from your browser. No dialog boxes and zero fuss.

Why do I need it?

h2> Line of business web applications are the norm, and a solid user experience is essential. For security reasons none of the top 5 browsers support direct printing, so users will always have to work through print dialogs EVERY time they print from your application. iPrint.Direct improves the user experience by reducing the number of 'clicks' that are required to print labels and documents. Once your label has been created it can be send down to the customers computer using only JavaScript - and this means your users don't have to click anything at all. Win!

How does it work?

iPrint.Direct installs a small application on your computer that can acts as a middle-man between your browser application and your printer. This allows your web application to send a print job directly to your printer, using only JavaScript. The communication happens between the browser and the computer hosting the installed application. This means that you don't even need to install the application on your computer - it can be anywhere on your network.

FAQ's

This sounds like Voodoo? : Like all great products iPrint Direct solves the printing problem in a very simple way. This can be integrated into any web application on any technology. Also the source code is open-source so you can inspect it if you wish. Is it secure? : Yes. Within the installed application you create a secret key, and then share this key with only the applications that are allowed to use your printers. The printers are only exposed to web applications running on your computer. Are my documents / labels uploaded to the iPrint.Direct servers? : No. Your documents and labels are sent directly from your browser to the locally installed application then to your printer. We save diagnostic information, but not document contents or data. The document printing process is completed wholly on-premise. Is it quick? : Yes. Very quick. Print speeds are usually only limited by the speed at which you can create your label, and the page speed of your printer. In testing we have achieved over 500 page per minute. Does my printer need to be connected to my computer? : Yes. We support all directly connected and networked printers. If you can see the printer in your Printer Settings screen, you can connect to it via iPrint.Direct. Do you support Networked Printers? : Yes. If you can see the printer in your Printer Settings screen, you can connect to it via iPrint.Direct. Can I print using an iPrint.Direct service on another computer? : Yes. You will need to ensure that the browser (where you create your labels) can communicate with the iPrint.Direct service on the printing computer. What documents formats do you support? : We support PDF and RAW printing. RAW is recommended for speed. Do you support label printing? : Yes. We support PDF and RAW ZPL and EPL label printing. We recommend ZPL as the fastest way to print shipping and return labels. Do you support different document sizes? : Yes. You can print any document size through the iPrint.Direct service - this is only limited by your printer - you can't print an A1 poster on an A4 device ;-) ---
